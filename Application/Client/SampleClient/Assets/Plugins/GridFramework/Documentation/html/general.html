Grids In General

In this chapter we will take a broad overview of the grids in Grid Framework and how they fit together. All grids are inside the GridFramework.Grids namespace.

In order to use a class in your namespace you have to explicitly import that namespace. In C# you need to put the following line at the head of your script:

using GridFramework.Grids;

If you are using UnityScript the line is as follows:

@import GridFramework.Grids;

Namespaces are used to group classes into categories and avoid name collisions. There can be different classes with the same name as long as they are in different namespaces. Grid Framework has also separate namespaces for renderers, extension methods and support for third-party plugins as well. We will cover those in their own chapters.

Each grid has a number of defining properties (such as Spacing and Shearing for rectangular grids) and coordinate conversion methods. They also define types for every coordinate system and events that get triggered when the properties of a grid change.

Additional functionality for the different grid types are provided as official extension methods. These are methods that are not strictly tied to the act of "being a grid", so they don't belong into the class itself, and they can be implemented without performance overhead using just the public API exposed by the grid classes.

The Grid class

This is an abstract class, meaning that all grids have to inherit from it. It does not expose any public API, only protected methods that need to be implemented when writing your own grids. Since there is no public API exposed it is irrelevant to end users.

The LayeredGrid class

Layered grids are 2D grids that are stacked on top of each other as "layers" to extend them into the third dimension. Unlike Grid, LayeredGrid does in fact expose public API: the Depth is the distance between two layers measured in world units and there are public events that go along with the depth; the vector Forward is a vector from one layer to another.

Layered grids are abstract and hexagonal and polar grids are concrete implementations of the concept. Layered grids are embedded in the XY-plane and you have to rotate their object if you want to orient them differently.

The RectGrid class

Rectangular grids are your standard cuboid boxes, but they can also be sheared into a parallelepiped shape. Parallelepiped would be better name, but the name of rectangular grids was their original name, so it has stuck.

The SphereGrid class

Spherical grids consist of concentric spheres that are sub-divided by parallel circles are meridian lines, similar to geographic coordinates on a planet's surface.

The PolarGrid class

Polar grids are concentric circles that are sub-divided into sectors. Being layered grids they actually form a cylinder, but the name "polar grid" has stuck, similar to the name of rectangular grids.

The HexGrid class

Hexagonal grids are layered grids as well and consist of regular hexagons that can be arranged in a number of ways. They have to most coordinate systems and renderers out of any grid.
